### 迭代：
新链表和旧链表的交接仪式
当前要交接的元素有两个信息：① 自己的信息(head) ② 队友的信息（head.next）

1 保存队友信息
ListNode temp = head.next;


2 接管新队友
head.next = newHead;

3 和新老板打招呼
newHead = head;


4 旧老板接管剩余队友 
head = temp;

交接仪式进行到什么时候结束呢？ 当然是把所有队友都交出去，即 head == null
while(head != null)

```
/**
 * Definition for singly-linked list.
 * public class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode(int x) { val = x; }
 * }
 */
class Solution {
   public ListNode reverseList(ListNode head) {
        // 一个新的空链表
        ListNode newHead = null;

        // 只要旧链表还有没交接的成员
        while (head != null){
            // 1 离队之前保存队友信息
            ListNode temp = head.next;
            // 2 接管新队友
            head.next = newHead;
            // 3 和新老板打招呼
            newHead = head;
            // 4 旧老板接管剩余队友
            head = temp;
        }
        // 返回新链表 
        return newHead;
    }
}
```

递归怎么想呢？

1 兄弟们，我不知道你们要走多远，但我会告诉你们一个鬼门关，你们看到了他，就往回返

2 兄弟们，交给你们去找鬼门关了！我在最后一道门口等你们，不过呢，你们从鬼门关出来，我会告诉你们一些事，我会在我的层次解决问题，下面我会说给你们听，你们也是类似，我示范给你们看

3 假设你们已经出色的完成了任务，好了，作为老板的我该做最后一步代表性处理工作了

```java
public ListNode reverseList(ListNode head) {
    // 我告诉你们鬼门关长啥样 —— 很简单的，就是这个链表没有元素，或者只有一个元素,这时我是不需要雇佣你们的
    if(head == null || head.next == null) return head;

    // 如果不是，那你们去找鬼门关吧
    ListNode newHead = reverseList(head.next);

    // 好，假设你们找到了鬼门关，你们看我怎么做，你们类似
    head.next.next = head;
    head.next = null;
    return newHead;
}
```
