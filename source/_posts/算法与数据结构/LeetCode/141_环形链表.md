给定一个链表，判断链表中是否有环。

为了表示给定链表中的环，我们使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环。

示例 1：
![示例1](https://assets.leetcode-cn.com/aliyun-lc-upload/uploads/2018/12/07/circularlinkedlist.png)

```
输入：head = [3,2,0,-4], pos = 1
输出：true
解释：链表中有一个环，其尾部连接到第二个节点。
```

## 1 分析条件
1 给定一个链表指针 head,指向第一个元素

2 那个 pos 怎么处理？ 不是输入呀,程序中肯定用不到的，仅仅是为了表示而已

## 2 确定问题

确定是否有环等价于—— 跑道上有两个指针，一个快，一个慢，假设快在前，慢在后，则问题等价于——如果有环，相当于环形跑道问题，快者必在某一时刻追上慢者，如果没环，快者先到达 null 

关键词：快慢指针

## 3 独立思考分析：（前期最长3分钟！！！）
```java
public boolean hasCycle(ListNode head) {
    // 如果仅有一个节点，或者没有节点，视为无环
    if (head == null || head.next == null) return false;// 下面 至少有 2 个节点

    ListNode slow = head;
    ListNode fast = head.next;

//        // 开始跑步 ,慢的，一次 1 步，快的，一次 2 步，如果快的先到达 null ,无环；如果快慢相遇（slow == fast）,有环
//        slow = slow.next;
//        fast = fast.next.next;

    // 跑多久呢？ 怎么控制呢?
    while (true) {
        // 开始跑步 ,慢的，一次 1 步，快的，一次 2 步，如果快的先到达 null ,无环；如果快慢相遇（slow == fast）,有环
        // TODO  空指针异常
        if (fast.next != null){ // 快的 要先判断能不能走，有没有桥 , fast 起到 探路的作用
            fast = fast.next.next;
        }else { // fast 走不了，中间的桥断了 ，提前 null
            return false;
        }
        slow = slow.next;

        // 这两句有没有先后顺序？ 如果有终点，一定是 fast 先到，所以先判断，是否到了终点
        if (fast == null) return false;
        if (fast == slow) return true;
    }
}

    
```

public boolean hasCycle2(ListNode head) {
    if(head == null || head.next == null) return false;

    ListNode slow = head;
    ListNode fast = head.next;

    // TODO 判断是有先后顺序的，否则会出现 空指针异常
    while( (fast != null) && (fast.next != null)){
        if(fast == slow) return true;
        slow = slow.next;
        fast = fast.next.next;
    }
    return false;
}
